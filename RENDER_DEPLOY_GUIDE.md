# 🚀 Creierosu Bot - Render & Neon Deployment Guide

This guide describes how to deploy the' Creierosu Telegram Bot and its FastAPI Dashboard using **Render** (as a' web service) and **Neon** (as the' PostgreSQL database).

## 1. Prerequisites
- A' **Neon.tech** account for the' database.
- A' **Render.com** account for the' web service.
- **Bot Token** from @BotFather.

## 2. Neon Database Setup
1. Create a' new project in Neon.
2. Copy the' **Connection String** (it starts with `postgresql://`).
3. Ensure it looks like: `postgresql://neondb_owner:PASSWORD@hostname/neondb?sslmode=require`.

## 3. Render Deployment
1. **Source Code**: Connect your GitHub repository to Render.
2. **Service Type**: **Web Service**.
3. **Runtime**: **Python**.
4. **Build Command**: `pip install -r requirements.txt`.
5. **Start Command**: `python main.py`. (Standardized port: 8080).

### Environment Variables
Set the' following in Render's **Environment** tab:
- `NEON_DB`: Your Neon connection string.
- `BOT_TOKEN`: Telegram bot token.
- `ADMIN_IDS`: Comma-separated Telegram IDs (e.g., `1234567,9876543`).
- `DASHBOARD_PIN`: PIN for logging into the' web dashboard (e.g., `1234`).
- `KEEP_ALIVE_URL`: Set this to your Render service URL (e.g., `https://my-service.onrender.com/`). This ensures it won't spin down!
- `PORT`: `8080`.

## 4. Seeding the' Database
Once you deploy, the' bot will automatically call `init_db()` and seed the' initial categories and items if the' database is empty.
To' **RESET and re-seed** the' database manually, run:
```bash
python reset_db.py
```

## 5. Maintenance
- The' **Dashboard** is available at your Render URL.
- Logs are available in the' Render **Logs** tab.
- Databases can be managed via the' **Neon Console**.
